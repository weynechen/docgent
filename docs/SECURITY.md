# SECURITY

## 当前边界

- 默认不上传本地全文到云端
- 仅在用户触发 AI 请求时发送选区及必要上下文
- 当前开发环境仍通过本地 Node 原型转发模型请求
- 长期目标是由云端 Python backend 统一承接鉴权、任务编排与 LangChain 调用

## 风险点

1. 模板式 backend 落地后，需要明确 API、service、repo、agent 之间的权限边界
2. 图片上传能力若保留，需要增加文件类型与大小约束审查
3. 本地文件系统接入后，需要避免误覆盖和路径越权

## 当前原则

1. AI 请求必须最小化上下文
2. 用户应知道哪些内容会发给模型
3. 不在 UI 中提前暴露超出当前安全边界的承诺
4. agent 只能返回建议，不直接写回正文
5. 前端最终只应调用 backend API，不直接访问 agent 执行层
