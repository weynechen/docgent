# SECURITY

## 当前边界

- 默认不上传本地全文到云端
- 仅在用户触发 AI 请求时发送必要上下文；从只发选区扩展到可按任务发送当前文档与会话信息
- 当前模型请求已由模板化 Python backend 统一承接
- 鉴权、会话与任务编排能力已随模板后端进入仓库，但 rewrite API 目前仍按匿名本地开发路径使用

## 风险点

1. rewrite API 当前未接入鉴权，需要在进入多用户或云部署前补齐权限边界
2. 图片上传能力若保留，需要增加文件类型与大小约束审查
3. 本地文件系统接入后，需要避免误覆盖和路径越权
4. `WebSearch` 会引入外部内容与时效性风险，必须保留来源和调用边界
5. `Write`、`Glob`、`Grep` 等工具若直接暴露给 agent，需要限制作用域到当前 workspace

## 当前原则

1. AI 请求必须最小化上下文
2. 用户应知道哪些内容会发给模型
3. 不在 UI 中提前暴露超出当前安全边界的承诺
4. agent 可以获得受控写入能力，但写入边界、作用域和冲突处理必须清晰可见
5. `WebSearch` 结果应可追踪来源，不能作为无来源事实写回
6. 前端最终只应调用 backend API，不直接访问 agent 执行层
