# 核心理念

## 1. 文档是工作对象，不是聊天输入框

写作对象必须有结构、阶段和版本，AI 只是协助修改，不是主导生成。

## 2. 智能体优先，但人保持控制

系统要围绕“指令 -> 对话 -> 工具调用 / 建议 / 写入 -> 审查或确认”工作，而不是黑盒式整篇替换。选区是可选上下文，不再是唯一入口。所有 AI 修改都应该可审查、可拒绝、可撤销，受控写入也必须有明确边界。

## 3. 记录系统优先于记忆

计划、设计决策、质量评估、技术债都应写入仓库中的 `docs/`，使后续智能体能在低上下文条件下继续工作。

## 4. 版本是产品语言，不是 Git 术语

用户理解的是“保存版本”“恢复版本”，不是 commit、stash、rebase。底层实现可以演进，但界面语言要稳定。

## 5. UI 优先服务正文

编辑器是视觉中心；左栏提供仓库/文档感知；右栏承载持续协作的 AI Chat、工具步骤与任务结果，但不与正文争抢注意力。

## 6. 小步验证优于大而全

优先验证写作主链路：打开文档、编辑、与 AI 多轮协作、审查或应用修改、保存、保存版本。平台优化、发布、Git 深度集成都应后置。
