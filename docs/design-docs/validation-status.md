# 设计验证状态

## 已验证

- 三栏工作台对写作主流程是可用的
- 标签栏 + 侧栏折叠/调宽可以改善工作区密度
- 右侧 AI 面板去噪后，主写作区域更聚焦
- 官方 Tiptap 模板比手写简化编辑区更适合真实编辑
- 右侧栏保留真实对话区和输入区，而不是说明卡片与快捷入口，这个方向成立
- notebook 作为顶层工作单元的心智可落地，draft 与 note 共存的侧栏结构可用
- 自动保存 + IndexedDB 本地缓冲可以支撑低延时编辑，并为断网恢复提供基础
- AI Chat 绑定 active notebook item 后，能够在真实右栏里完成读写回环而不是停留在说明态

## 部分验证

- 版本快照心智已形成，但恢复和富文本同步仍需更多回归验证
- AI 改写流程已接入真实模型通道，但还没有系统质量验证和 prompt 回归基线
- 右侧 AI Chat 作为高频协作入口的方向已经明确，但多轮对话质量、长会话稳定性和执行可见性尚未完成系统验证
- Markdown 往返可用，但复杂格式下的稳定性还没有系统验证
- 桌面端与云端服务的长期目录边界已明确，但模板式 Python/LangChain 后端方案尚未落地验证
- notebook 冲突态已在数据模型和状态栏中预留，但用户可恢复的冲突处理体验还未完成验证

## 未验证

- 本地目录打开与文件树
- 外部文件冲突处理
- Git 深度集成
- 长文档与大纲级结构编辑性能
- 外链资料与文件导入后的 notebook 扩展模型
- agent 工具集在真实 notebook/workspace 上的安全边界与可用性
