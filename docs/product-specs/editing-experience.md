# 编辑体验规格

## 目标

让用户感觉自己在使用一个真正可写、可改、可回退的编辑器，而不是嵌了聊天框的页面原型。

## 当前要求

- 正文排版紧凑，减少大面积留白
- 标题层级明显，但不过度夸张
- 分割线和工具条分隔视觉弱化
- 侧栏不应挤压正文到不可读宽度
- 保存反馈应轻量，不遮挡输入
- 右侧 AI Chat 的 assistant 消息需要流式更新，不能等 agent 全部完成后才一次性显示

## 编辑器约束

- 使用 Tiptap 官方 `simple-editor` 模板作为基础
- 保持模板源码在仓库内，便于后续裁剪
- 产品改写逻辑由业务层控制，不把 AI 能力耦合进模板内部
- 后端临时工作区是文档唯一真相源，前端只作为远程编辑与浏览视图
- MVP 中人工编辑采用整文保存，不实现前后端双向 patch 同步
- AI 改写由后端读取完整 Markdown 并返回候选文档版本，前端只展示 diff 并确认应用
- 右侧 AI Chat 是工作台的一等交互区，不再只是选区改写附属面板
- assistant 消息、工具步骤和执行状态需要在右栏持续流式可见

## 后续优化方向

1. 裁剪官方模板中不必要的按钮和扩展
2. 收敛真正需要的节点样式
3. 增强中文写作场景下的默认排版
4. 将右侧 AI 面板从单次结果展示升级为持续流式对话体验
